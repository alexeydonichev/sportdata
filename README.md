# SportData Platform

Аналитическая платформа для мультимаркетплейс-продавца спортивных товаров.
Агрегация данных с Wildberries, Ozon, Яндекс Маркет в единый дашборд.

## Архитектура

| Сервис | Стек | Порт | Назначение |
|--------|------|------|------------|
| frontend | Next.js 15, React 19, TailwindCSS | 3000 | UI + BFF API routes |
| api-gateway | Go 1.23, PASETO, bcrypt | 8080 | Auth, RBAC, admin API |
| etl-service | Go 1.23 | - | Синхронизация с маркетплейсами |
| postgres | PostgreSQL 16 | 5432 | Основная БД |
| redis | Redis 7 | 6379 | Кеш, rate limiting |

## Быстрый старт

1. Скопировать переменные окружения и заполнить реальные значения:

    cp .env.example .env

2. Запустить:

    docker compose up -d

3. Применить миграции (при первом запуске автоматически):

    make migrate

4. Создать суперадмина:

    make admin

5. Открыть http://localhost:3000

## Структура проекта

    sportdata/
    ├── migrations/              # Единая точка миграций
    │   ├── 001_schema.sql       # Основная схема
    │   ├── 002_seed.sql         # Тестовые данные (30 товаров, 90 дней продаж)
    │   └── 003_etl_extensions.sql
    ├── api-gateway/             # Go — авторизация и админ-API
    ├── etl-service/             # Go — ETL worker
    ├── frontend/                # Next.js — UI и BFF
    ├── docker-compose.yml
    ├── Makefile
    └── .env.example

## Makefile команды

| Команда | Описание |
|---------|----------|
| make up | Запуск всех сервисов |
| make down | Остановка |
| make logs | Логи всех сервисов |
| make migrate | Применить миграции |
| make admin | Создать суперадмина |
| make psql | Подключиться к БД |

## Дашборд

- Главная — выручка, прибыль, заказы, средний чек
- Продажи — таблица с фильтрами по МП, категории, дате
- Товары — каталог, карточка товара с аналитикой
- Склад — остатки по маркетплейсам и складам
- Аналитика — P&L, ABC-анализ, юнит-экономика, география, возвраты
- Синхронизация — управление API-ключами, история ETL
- Админ — RBAC, управление пользователями

## Безопасность

- Аутентификация: PASETO v4 токены
- Пароли: bcrypt
- API-ключи маркетплейсов: AES-256 шифрование в БД
- RBAC: 5 уровней ролей (super_admin -> manager)
- Rate limiting: Redis
- Audit log: все действия пользователей
